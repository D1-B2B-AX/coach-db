# 이메일 알림 설정 가이드

섭외 요청, 코치 수락/거절, 투입 확정/취소 시 이메일이 자동 발송됩니다.
설정하지 않아도 기존 기능(인앱 알림, 웹 푸시)은 정상 동작합니다.

## 1. 공용 Gmail 계정 준비

1. 새 Gmail 계정 생성 (예: `coach-noreply@gmail.com`)
2. [myaccount.google.com](https://myaccount.google.com) → 보안 → **2단계 인증** 켜기
3. 2단계 인증 설정 페이지 하단 → **앱 비밀번호** 클릭
4. 앱 이름 입력 (예: "코치관리") → **만들기**
5. 생성된 16자리 비밀번호 복사

## 2. 환경 변수 추가

Railway (또는 `.env.local`)에 아래 두 값을 추가합니다:

```
GMAIL_USER=coach-noreply@gmail.com
GMAIL_APP_PASSWORD=xxxx xxxx xxxx xxxx
```

## 3. 동작 확인

환경 변수 설정 후 재배포하면 자동 적용됩니다.
테스트: 매니저 페이지에서 코치 섭외 → 해당 코치 이메일로 수신 확인.

## 알림 종류별 이메일

| 이벤트 | 수신자 | 제목 형식 |
|--------|--------|-----------|
| 섭외 요청 | 코치 | [섭외 요청] 김매니저 — 2026-04-10 |
| 코치 수락 | 매니저 | [코치 수락] 박코치 — 2026-04-10 |
| 코치 거절 | 매니저 | [코치 거절] 박코치 — 2026-04-10 |
| 투입 확정 | 코치 | [투입 확정] 2026-04-10 과정명 |
| 투입 취소 | 코치 | [투입 취소] 2026-04-10 |

- 코치에게 보내는 메일에는 매니저 이메일이 reply-to로 설정됩니다 (답장하면 매니저에게 감)
- 코치 이메일이 DB에 없으면 해당 코치는 이메일 skip됩니다
