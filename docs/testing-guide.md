# 코치DB 테스트 가이드

> 스테이징: https://coach-db-staging.up.railway.app
> 테스트 일자: 2026-03-31 (월)

---

## 1. 로그인

- [ ] 스테이징 URL 접속
- [ ] 구글 계정으로 로그인 (`@day1company.co.kr`)
- [ ] 대시보드가 보이면 성공

> 로그인 안 되면 어드민에게 매니저 등록 요청

---

## 2. 대시보드 (`/dashboard`)

- [ ] 달력에서 오늘 날짜 클릭 → 하단에 코치 목록 표시
- [ ] 날짜 범위 선택 (미래 날짜 2개 클릭) → 해당 기간 코치 표시
- [ ] 월 이동 (< >) 버튼 동작
- [ ] 분야/근무유형 필터 선택 → 목록 필터링
- [ ] 동기화 버튼 클릭 → 토스트 메시지 확인

---

## 3. 코치 목록 (`/coaches`)

- [ ] 코치 이름으로 검색 (예: "김")
- [ ] 분야 필터 선택 → 목록 필터링
- [ ] 정렬 변경 (이름순 / 근무일 많은 순 / 평가 높은 순)
- [ ] 코치 여러 명 체크 → 이메일 내보내기 → 엑셀 다운로드
- [ ] 코치 이름 클릭 → 상세 페이지 이동

---

## 4. 코치 상세 (`/coaches/{id}`)

- [ ] 프로필 탭: 기본 정보 표시 확인
- [ ] 스케줄 탭: 달력 + 일정 표시
- [ ] 근무이력 탭: 투입 이력 목록 표시
- [ ] 상태 변경: 활동중 ↔ 비활동 토글
- [ ] 비활동 전환 시 복귀 예정월 입력
- [ ] 상태 메모 클릭 → 메모 입력 → 저장
- [ ] 수정 버튼 → 편집 페이지에서 정보 수정 → 저장

---

## 5. 코치 등록 (`/coaches/new`)

- [ ] "+ 등록" 클릭 → 등록 폼
- [ ] 이름만 입력 후 저장 → 성공
- [ ] 생성 완료 시 코치 고유 링크 표시 + 복사 버튼
- [ ] 등록한 코치가 목록에 보이는지 확인

> 테스트 후 등록한 코치는 삭제해주세요

---

## 6. 스케줄 입력 → 대시보드 반영 (End-to-End)

이 테스트가 핵심입니다. 코치 입장에서 스케줄을 입력한 후, 매니저 화면에서 제대로 보이는지 확인합니다.

### Step 1: 코치로 스케줄 입력

- [ ] 위 링크 접속 → 전화번호 확인 → 일정 화면 진입
- [ ] **4월 7일(월)** 클릭 → 09:00~12:00 시간대 선택
- [ ] **4월 8일(화)** 클릭 → 13:00~18:00 시간대 선택
- [ ] **4월 9일(수)** 클릭 → "불가" 버튼으로 전체 불가 처리
- [ ] 저장 버튼 클릭 → "저장됨" 표시 확인
- [ ] 프로필 수정 버튼 → 아무 필드나 변경 → 저장

### Step 2: 매니저로 대시보드 확인

코치 입력을 마친 후, 매니저 계정으로 돌아와서:

- [ ] 대시보드 → 4월 7일 클릭 → 방금 입력한 코치가 목록에 보임
- [ ] 4월 8일 클릭 → 해당 코치가 오후 시간대로 표시
- [ ] 4월 9일 클릭 → 해당 코치가 목록에 안 보임 (불가 처리)
- [ ] 코치 상세 → 스케줄 탭 → 입력한 일정이 달력에 표시
- [ ] 대시보드 상단 상태 요약 → "입력완료" 수가 늘어남

---

## 7. 코치 화면 추가 확인

- [ ] 월 이동 → 이전 달은 조회만, 다음 달은 편집 가능
- [ ] 확정된 투입 일정(있는 경우) → 시간대 잠금, 수정 불가
- [ ] 모바일에서 접속 → 레이아웃 깨지지 않는지

---

## 8. 어드민 (`/admin`) — 관리자만

- [ ] 관리자 탭: 매니저 목록 + 권한 변경

- [ ] 동기화 탭: 수동 동기화 실행 + 로그 확인
