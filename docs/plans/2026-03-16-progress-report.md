## 구현 내역

### 1. 노션 연동 강화

2025/2026 두 개 노션 DB를 동시에 연동하여 코치 데이터를 보강했다.

| 항목 | 내용 |
|------|------|
| 2026 DB | 근무 유형(실습코치/운영조교), 유형(삼전 DX/DS) 가져오기 |
| 2025 DB | 유형, 활동 상태(활동중/미활동중/탈락) 매핑 |
| 활동 여부 | 2026 DB에 있으면 available, 없으면 unavailable |
| subjects 합산 | 시트 + 2026 + 2025 union (덮어쓰기 없이 합치기) |
| 노션 전용 코치 | DB에 없는 코치 21명 자동 생성 |
| 연락처 보완 | 시트에 없으면 노션 2025 DB에서 채우기 |
| 버그 수정 | 시트 upsert 시 portfolio_url/notion_url이 매번 null로 덮어쓰이던 문제 해결 |

### 2. 데이터 품질 개선

| 항목 | 내용 |
|------|------|
| is_new 제거 | 구글시트 B열 기반 판단이 부정확하여 전체 폐기 |
| 취소 행 필터링 | A열/H열에 "취소" 포함 행 제외, subjects에서도 "취소" 제거 |
| 연락처 파싱 | "이메일/전화번호" 혼합 형태 분리 처리 |
| 동명이인 병합 | 2명 이하면 연락처 없어도 병합 |

### 3. 메인 테이블 UI

| 항목 | 내용 |
|------|------|
| 활동 여부 | 초록/회색 도트로 테이블 최좌측 표시 |
| 구분 | 필드별 색상 (실습코치/운영조교/삼전DX/DS/보조강사/멘토) |
| 활동연도 | 드롭다운 (2026 외 3년 ▾) |
| 연락처 | 전화/이메일 아이콘 + 텍스트 |
| 가능 분야/커리큘럼 | ResponsiveChips — 너비에 맞게 자동 +N 표시 |
| 정렬 | 활발한 활동순 (이력수 + 연속연수 + 최근연도 가중치) + 미활동자 항상 하단 |
| 패널 열림 시 | 커리큘럼 컬럼 자동 숨김, 황금비(62:38) 분할 |

### 4. 필터 시스템

| 필터 | 방식 |
|------|------|
| 구분 | 멀티셀렉트 드롭다운 (파란색) |
| 연도 | 멀티셀렉트 드롭다운 (회색) |
| 분야 | 멀티셀렉트 드롭다운 (앰버) |
| 커리큘럼 | 멀티셀렉트 드롭다운 (보라) |
| 전체 | 모든 드롭다운에 전체선택/선택해제 버튼 |

### 5. 검색

- 필터 바에 검색 입력란 추가
- 검색 대상: 이름, 전화번호, 이메일, 사번, 소속, 생년월일, 특이사항, 근무 가능 세부, 구분, 가능 분야, 가능 커리큘럼, 과정명
- 검색어 하이라이트 (노란색)

### 6. 상세 패널 UI

| 항목 | 내용 |
|------|------|
| 연락처 | 아이콘 바로 통합 (전화/이메일 클릭 복사, Notion/포트폴리오 새 탭) |
| 메타 정보 | 소속(건물 아이콘), 생년월일(캘린더 아이콘) |
| 섹션 순서 | 캘린더 → AI 분석 키워드 → AI 이력 분석 → 가능 커리큘럼 → 근무 이력 → 특이사항 → 메모 |
| 아이콘 | 커리큘럼(학모자), 근무이력(서류가방), 메모(말풍선), 동기화(새로고침+회전) |

### 7. AI 분석 키워드

GPT를 활용하여 코치의 기술 키워드를 자동 추출하고 빈도순으로 표시한다.

| 항목 | 내용 |
|------|------|
| 키워드 소스 | 과정 이력 + 포트폴리오 PDF (Google Drive에서 다운로드 → pdf-parse) |
| 가중치 | AI 추출 ×2, 노션 skill_stack ×1 |
| 정규화 | 공백/&// 차이 무시 (동의어 합산) |
| 프롬프트 | 과정명 키워드 필수 추출, R/AI 오매칭 방지, 중복 표기 방지 |
| 자동 실행 | 상세 패널 열 때 ai_summary 없으면 자동 GPT 호출 |
| 캐싱 | portfolio_url 변경 시만 ai_summary 초기화, 나머지는 캐시 유지 |
| 산출 기준 | ? 아이콘 클릭 시 팝업으로 가중치 설명 |

### 8. 캘린더

| 항목 | 내용 |
|------|------|
| 위치 | 상세 패널 최상단 |
| 데이터 | coach_schedules(상세 일정) + courses(과정 시작~종료일) 합산 |
| 표시 | 날짜 셀 배경색, 겹침 시 대각선 그라데이션 |
| 범례 | 미래 과정만 표시, 1개면 텍스트/2개 이상이면 칩 |
| 기타 | 오늘 파란 원, 과거 흐리게, 근무 가능 배너 |

### 9. 스케줄 시트 연동

외부 Google Sheets의 정형 일정 데이터를 코치별 캘린더에 반영한다.

| 항목 | 내용 |
|------|------|
| API | `POST /api/sync-schedule` (fileId + label 파라미터) |
| 파싱 | 주차/시작일/종료일/과정명/코치명 컬럼, "26년 일정" 탭 자동 선택 |
| 복수 배정 | "박요한 / 오찬빈" → 각각 스케줄 생성 |
| 라벨 | 시트 과목명 대신 고정 라벨 지정 가능 (예: "삼성전자 - SW학부 교육과정") |
| 미매칭 코치 | 자동으로 DB에 생성 후 매칭 |
| DB | coach_schedules 테이블 (coach_id, date, label) |
| 실적 | 삼성전자DS SW학부: 30행 → 111건 스케줄, 10명 코치 |

## 현재 만들어진 것

**화면 3개** (변경 없음):
- `/login` — 구글 로그인
- `/` — 코치 목록 + 필터 + 검색 + 상세 패널 (캘린더/AI 키워드 포함)
- `/coaches/[id]` — 코치 상세 (모바일)

**API 3개** (+2):
- `POST /api/sync` — 구글시트 + 노션 → DB 동기화
- `POST /api/coaches/[id]/summary` — AI 키워드 추출 (GPT)
- `POST /api/sync-schedule` — 스케줄 시트 → coach_schedules 동기화

**DB 테이블 8개** (+1):
- coaches, courses, coach_courses, coach_memos, sync_logs (기존)
- coach_schedules (신규)
- instructors, instructor_courses (3차 대비)

**신규 컴포넌트 4개:**
- CoachCalendar — 월별 캘린더
- FilterDropdown — 멀티셀렉트 필터
- ResponsiveChips — 반응형 칩 (+N)
- MergedKeywords — AI 분석 키워드 (CoachPanel 내)

**커밋 8개** (이번 세션), 총 18개.
