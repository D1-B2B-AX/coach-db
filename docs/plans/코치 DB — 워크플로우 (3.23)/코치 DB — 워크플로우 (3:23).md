## 5. 노드별 시스템 매핑

### 설계문서 #1로 대체되거나 불필요해지는 단계

- 아는 코치가 그 과정을 소화할 수 있는지 생각해본다 → 코치 상세 페이지에서 확인
- 삼전코치 일정표 시트를 연다 → 대시보드 캘린더 뷰
- 그 날 일정이 되는 코치를 찾는다 → 날짜 클릭 시 가능 코치 자동 표시
- 동료에게 아는, 괜찮은 코치가 있는지 묻는다 (x2) → 대시보드 필터로 직접 검색
- 있다고 하면 → 동료에게 묻는 단계 자체가 사라짐

### 설계문서 #2로 대체되거나 불필요해지는 단계

- 해당 코치에게 일정이 되는지 문자를 한다 (x2) → 배정 요청 보내기
- 일정이 되는 코치에게 메일을 보낸다 → 배정 요청 보내기
- 답을 기다린다 → 배정 요청 관리 페이지에서 현황 확인
- 해당코치가 일정이 된다면 → 구인 완료 (x2) → 수락 시 자동 확정
- 해당 코치가 일정이 안된다면 → 다른 코치에게 추가 요청
- 해당 코치가 승낙하면 → 구인 완료 → 수락 시 자동 확정
- 거절하면 → 다른 코치에게 추가 요청 또는 외부 구인
- 코치에게 메일을 보냈던 이력을 찾아 내용을 복사한다 → 배정 요청이 시스템화되므로 불필요
- 답이 안오면 문자를 보낸다 → 응답 기한 만료 시 자동 거절

## 6. 비교 요약

| 단계 | 기존 | 시스템 도입 후 | 설계문서 |
| --- | --- | --- | --- |
| 적합한 코치 찾기 | 기억 + 동료에게 질문 | 대시보드 필터로 직접 검색 | #1 |
| 일정 확인 | 시트 열어서 수동 확인 | 날짜 클릭 → 바로 표시 | #1 |
| 코치에게 연락 | 문자/메일 수동 발송 | 배정 요청 버튼 클릭 | #2 |
| 응답 대기 | 답 안오면 다시 문자 | 기한 만료 시 자동 거절 | #2 |
| 구인 확정 | 별도 기록 | 수락 시 자동 확정 등록 | #2 |
| 구인글 작성 | 이전 글 찾아서 복사 | 과정 정보 기반 자동 생성 | #3 |
| 신규 코치 등록 | 정보 처음부터 수동 입력 | 합격자 정보로 바로 등록 | #3 |

## 7. 개발 우선순위

| 순서 | 범위 | 효과 |
| --- | --- | --- |
| MVP | 설계문서 #1 — 스케줄 수집 + 코치 관리 | 코치 탐색/일정 확인 자동화. 동료에게 물어보는 단계 제거 |
| Post-MVP | 설계문서 #2 — 배정 요청/응답 | 코치 연락/응답 대기 자동화. 수동 문자/메일 제거 |
| Post-MVP 2 | 설계문서 #3  | 외부 구인 시 반복 작업 경감 |
