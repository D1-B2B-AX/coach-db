## 1. 자동 동기화 아키텍처

```
┌─────────────┐     POST /api/sync/engagements     ┌──────────────┐
│  캘린더 UI   │ ──── 새로고침 버튼 ──────────────→  │  Next.js API  │
│  (새로고침)  │                                     │   Route       │
└─────────────┘                                     │              │
                                                    │  ┌──────────┐│
┌─────────────┐     POST /api/sync/samsung-schedule │  │ 공통 sync ││   ┌────────────┐
│  관리자 UI   │ ──── 동기화 버튼 ─────────────────→ │  │  로직    ││──→│ Google     │
│  (삼성일정)  │                                     │  └──────────┘│   │ Sheets API │
└─────────────┘                                     └──────────────┘   └────────────┘
                                                           ↑
┌─────────────┐     POST /api/sync/samsung-schedule │
│ GitHub      │ ──── 매월 마지막주 화요일 ─────────→│
│ Actions     │     (Authorization: Bearer SECRET)
└─────────────┘
```

### API Routes

| Route | 용도 | 트리거 |
|---|---|---|
| `POST /api/sync/engagements` | 계약/일정 데이터 동기화 | 캘린더 새로고침 버튼 |
| `POST /api/sync/samsung-schedule` | 삼성 스케줄 동기화 | 관리자 버튼 + GitHub Actions 월 1회 |

### 인증

- **UI 버튼**: 기존 세션 인증 (매니저만)
- **GitHub Actions**: `SYNC_API_SECRET` 환경변수 Bearer 토큰 검증

### 응답

동기화 결과 반환: 생성/수정/스킵/에러 건수 → UI에 토스트 표시

### GitHub Actions 스케줄

- 대상: 삼성 스케줄
- 주기: 매월 마지막 주 화요일 (삼성이 마지막 주 월요일에 업데이트)
- 방법: `POST /api/sync/samsung-schedule` + Bearer 토큰

## 2. 데이터 소스 오너십

### coaches 테이블

| 필드 | 소스 | 비고 |
|---|---|---|
| `name` | **구글시트** (E열) | 최초 생성 시, 이후 수동 수정 가능 |
| `email` | **구글시트** (N열) | 계약서 데이터 |
| `phone` | **구글시트** (O열) | 계약서 데이터 |
| `employeeId` | **구글시트** (D열) | 동기화 키 |
| `workType` | **노션** 우선, 구글시트 보완 | 노션에 없으면 시트 F열 |
| `portfolioUrl` | **노션** | 이력서 및 포트폴리오 |
| `selfNote` | **노션** | 특이사항/히스토리 |
| `availabilityDetail` | **노션** | 근무 가능 기간 |
| `availability` | **앱 수동 입력** | available/unavailable/unknown (D-029) |
| `hiredBy` | **구글시트** (G열) | 담당 매니저 |
| `fields` | **노션** | coach_fields 테이블 |
| `curriculums` | **노션** | coach_curriculums 테이블 |

### engagements 테이블

| 필드 | 소스 | 비고 |
|---|---|---|
| `coachId` | **구글시트** (E열) | 이름으로 coaches 조회 매칭 |
| `courseName` | **구글시트** (H열) | |
| `startDate` | **구글시트** (J열) | |
| `endDate` | **구글시트** (K열) | |
| `hourlyRate` | **구글시트** (I열) | 과정별 시급 — 근무이력에서 표시 |
| `hiredBy` | **구글시트** (G열) | 담당 매니저 |
| `workType` | **구글시트** (F열) | 해당 계약의 근무유형 |
| `status` | **계산** | startDate/endDate 기준 scheduled/in_progress/completed |

**중복 판단 키**: `coachId + courseName + startDate`

### engagement_schedules / coach_schedules 테이블

| 필드 | 소스 | 비고 |
|---|---|---|
| `date` | **구글시트** (M열 파싱 또는 C~D열 범위 전개) | |
| `startTime` / `endTime` | **구글시트** (M열 파싱) | 삼성은 고정 09:00~18:00 |
| `engagementId` | **계산** | 상위 engagement 연결 |

### 앱 전용 데이터 (외부 소스 없음)

| 테이블.필드 | 설명 |
|---|---|
| `coaches.availability` | 수동 입력 (D-029) |
| `coach_memos` | 매니저가 직접 작성 |
| `sync_logs` | 동기화 실행 시 자동 기록 |

## 3. 충돌 처리 / 업데이트 규칙

### 동기화 시 행동 원칙

| 상황 | 처리 |
|---|---|
| DB에 없는 코치 | **자동 생성** (2026년 계약 데이터인 경우) |
| DB에 있는 코치, 시트에 새 정보 | **덮어쓰기** (email, phone 등 시트 소유 필드) |
| DB에 있는 코치, 앱 수동 입력 필드 | **건드리지 않음** (availability, memos) |
| 같은 engagement 키 존재 | **스킵** (coachId + courseName + startDate 중복) |
| 시트 행에 "취소" 표시 | **스킵** (import 안 함) |
| 코치 이름 매칭 실패 | **로그 남기고 스킵** (sync_logs에 에러 기록) |

### 소스 간 우선순위 (같은 필드에 여러 소스)

| 필드 | 우선순위 | 이유 |
|---|---|---|
| `workType` (coaches) | 노션 > 구글시트 | 노션이 더 정확한 마스터 |
| `email`, `phone` | 구글시트 (최신 계약 기준) | 계약서가 최신 연락처 |
| `fields`, `curriculums` | 노션 단독 | 구글시트에 없음 |

### 삼성 스케줄 특이사항

| 상황 | 처리 |
|---|---|
| 재동기화 시 기존 삼성 일정 | **삭제 후 재생성** (전체 교체) |
| 코치 이름 "/" 구분 | 분리 후 각각 매칭 |
| 코스명 | 고정값 "삼성전자 SW학부 교육과정" |
| 시간 언급 없으면 | 고정 09:00~18:00 |

## 4. 동기화 주기 요약

| 대상 | 트리거 | 주기 | 방법 |
|---|---|---|---|
| 계약/일정 (engagements) | 캘린더 새로고침 버튼 | 수동 | `POST /api/sync/engagements` |
| 삼성 스케줄 | 관리자 페이지 버튼 | 수동 | `POST /api/sync/samsung-schedule` |
| 삼성 스케줄 | GitHub Actions | 매월 마지막 주 화요일 | 같은 API route 호출 |
| 노션 데이터 | CLI 스크립트 | 수동 (이번 범위 밖) | `npx tsx scripts/import-notion-*.ts` |
