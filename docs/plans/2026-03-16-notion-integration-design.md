# 노션 연동 설계

> 작성일: 2026-03-16

## 목표

노션 DB(실습코치/운영조교 DB)의 코치 상세 정보를 기존 코치 레코드에 매칭하여 동기화한다.

## 동기화 흐름

```
동기화 버튼 → 1) 구글시트 동기화 → 2) 노션 동기화 → 완료 토스트
```

- 구글시트가 코치 레코드를 생성/갱신하고, 노션이 추가 필드를 채우는 순서.
- 노션에만 있고 구글시트에 없는 코치는 생성하지 않되, 매칭 안 된 이름을 동기화 결과에 포함하여 사용자에게 알린다.

## 매칭 전략

노션 페이지의 `이름` + (`연락처` or `이메일`) → DB coaches 테이블 매칭.

- 이름 + 연락처 일치 → 매칭
- 이름 + 이메일 일치 → 매칭
- 매칭 안 됨 → 건너뛰고 이름을 응답에 포함

## 필드 매핑

### 기존 컬럼 업데이트

| 노션 필드 | 노션 타입 | → DB 컬럼 |
|-----------|----------|-----------|
| 가능 커리큘럼 | multi_select | `skill_stack` |
| 이력서 및 포트폴리오 | rich_text | `portfolio_url` |
| Notion page URL | 자동 | `notion_url` |
| 생년월일 | rich_text | `birth_date` |
| 소속 | rich_text | `organization` (구글시트에 없을 때) |

### 새 컬럼 추가

| 컬럼명 | 타입 | 노션 필드 | 용도 |
|--------|------|-----------|------|
| `available_fields` | `text[]` | 교육 및 가능 분야 | 대분류 필터 |
| `availability_detail` | `text` | 근무 가능 세부 내용 | 섭외 참고 |
| `notes` | `text` | 특이사항 / 히스토리 | 공유 메모 (읽기 전용) |

## UI 변경 (상세 패널)

- 신규/구분 배지 아래: **교육 및 가능 분야** 배지 추가
- **스킬 스택** 섹션 (가능 커리큘럼)
- 연락처 아래: **근무 가능 세부 내용** 텍스트
- 메모 위: **특이사항/히스토리** (읽기 전용, 노션에서 관리)

## 범위 밖

- 단방향 동기화만 (노션 → 앱). 앱에서 노션 수정 안 함.
- 전문 분야, 근무 가능 기간 필드는 제외 (중복/불필요).
