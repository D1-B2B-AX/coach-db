## 이번 작업 (3/17) 구현 내역

### 1. 비활성 코치 관리

| 항목 | 내용 |
|------|------|
| 기준 | 노션 2026 DB에 있는 코치만 available, 나머지 전부 unavailable |
| 노출 | 기본적으로 비활성 코치 숨김 (활성 코치만 표시) |
| 동기화 | sync 시 전체 unavailable 초기화 → 2026 DB 포함 코치만 available 재설정 |
| 노션 전용 코치 | 구글시트에 없어도 노션 2026 DB에 있으면 자동 생성 (21명 추가) |

### 2. 정렬 개선

| 항목 | 내용 |
|------|------|
| 1순위/2순위 | 두 개의 셀렉트로 정렬 기준 조합 가능 |
| 옵션 | 활동순, 바쁜 순, 이름순 |
| 중복 방지 | 1순위 선택 시 2순위에서 해당 옵션 자동 제외 |

### 3. 체크박스 선택 + xlsx 다운로드

| 항목 | 내용 |
|------|------|
| 체크박스 | 테이블 좌측에 행별 체크박스, 헤더에 전체선택 |
| 연락처 다운로드 | 선택된 코치의 이름 + 전화번호 → xlsx |
| 이메일 다운로드 | 선택된 코치의 이름 + 이메일 → xlsx |
| 선택 바 | 상단에 "N명 선택" + 다운로드 버튼 + 선택 해제 |

### 4. 검색 범위 확장

기존 6개 필드에서 전체 필드로 확장:

- 이름, 전화번호, 이메일, 사번, 소속, 생년월일
- 특이사항, 근무 가능 세부, 구분, 가능 분야, 가능 커리큘럼
- **과정명** (coach_courses → courses.name 조인)

### 5. 상세 패널 UI 정리

| 항목 | 내용 |
|------|------|
| AI 키워드 제거 | MergedKeywords 컴포넌트 삭제, 노션 skill_stack 기반 가능 커리큘럼만 표시 |
| 섹션 순서 | 가능 커리큘럼 → 일정 → 특이사항 → 근무 이력 → 메모 |
| 아이콘 추가 | 가능 커리큘럼(학사모), 특이사항(정보) |

### 6. 캘린더 UI 전면 개선

| 항목 | 내용 |
|------|------|
| 셀 표시 | 시간 직접 표시 (형광펜 스타일 배경색), 시간 없으면 과정명 축약 |
| 시간 포맷 | 09:00 → 9:00 (앞자리 0 제거) |
| 오버플로우 | 최대 2개 표시, 3개 이상이면 +N (클릭 시 전체 팝업) |
| 오늘 표시 | ring-1 ring-blue-400 (셀 테두리) |
| 과정 색상 | 칩과 겹치지 않는 orange/lime/yellow/pink 계열 10색, 전체 라벨 정렬 기반 배정 |
| 데이터 정합성 | schedules 데이터 있으면 courses 중복 제거 |
| 범례 | 과거 과정도 포함, 6개월 이전 이동 제한 |
| 버그 수정 | parseDate UTC 오프셋 (1월 과정 안 보이던 문제) |

## 현재 만들어진 것

**화면 3개:**
- `/login` — 구글 로그인
- `/` — 코치 목록 + 필터 + 검색 + 정렬 + 체크박스 + 상세 패널
- `/coaches/[id]` — 코치 상세 (모바일)

**API 4개:**
- `POST /api/sync` — 구글시트 + 노션 → DB 동기화
- `POST /api/coaches/[id]/summary` — AI 키워드 추출 (GPT)
- `POST /api/coaches/[id]/notion-content` — 노션 페이지 본문 조회
- `POST /api/sync-schedule` — 스케줄 시트 → coach_schedules 동기화

**DB 테이블 8개:**
- coaches, courses, coach_courses, coach_memos, sync_logs, coach_schedules
- instructors, instructor_courses (3차 대비)
