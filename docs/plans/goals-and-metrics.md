# 코치DB — 목표 및 측정 지표

## 왜 만드는가

매번 새로 구인 → 면접 → 온보딩 하는 대신, 이미 검증된 코치 풀에서 바로 매칭.
구인 비용과 리스크가 줄고, 배정 속도가 빨라지고, 코치 품질도 예측 가능해진다.

## 목표 체계

| 단계 | 목표 | 코치DB 역할 |
|------|------|------------|
| 궁극 | 매출 증대 | 간접 기여 |
| 중간 | 같은 인력으로 더 많은 과정 운영 | 병목 제거 |
| 직접 | 코치 풀 구축·유지로 외부 구인 없이 즉시 매칭 | 핵심 도구 |

## 기대효과 및 측정지표

| 기대효과 | 측정지표 | 측정방법 | 비고 |
|----------|---------|---------|------|
| 외부 구인 감소 | 월별 외부 구인 공고 건수 | 수동 기록 | 도입 전후 비교 |
| 배정 속도 향상 | 코치 필요 시점 → 확정까지 소요 시간 | 설문 (월 1회) | 체감 시간 |
| 풀 전체 활용 | 매니저당 배정한 코치의 명수 (다양성 체크) | 앱 자동 (engagement 데이터) | 특정 코치 편중 감소 |
| 유휴 코치 감소 | 3개월 이상 미매칭 코치 비율 | 앱 자동 | 풀 효율성 |
