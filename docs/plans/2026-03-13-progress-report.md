## 1단계: 설계 (Clarify)

스펙을 기반으로 3가지를 확인한 뒤 설계를 확정했다.

| 확인 사항 | 결정 |
|-----------|------|
| 동기화 키가 뭔지 | 구글시트 D열 사번 |
| 교육 과정 데이터 출처 | H열 과정명 (일단 이름만, 나머지는 나중에) |
| 한 코치가 여러 과정에 있을 때 시트 구조 | 같은 사번이 여러 행에 등장 |

decisions.md와 대조하여 설계 문서의 오류 3건도 수정했다:

- `is_available` boolean → `availability` 3상태 (가능/불가/미확인)
- `skill_stack` 필드 누락 → 추가
- 과정 동기화 범위 → 과정명 + 고용기간 포함

## 2단계: 문서 구조 세팅

Clarify ↔ Build 반복을 추적하기 위한 문서 체계를 만들었다.

```
coach/
├── CLAUDE.md                    ← 매 세션 자동 로드 (현재 상태 + 문서 위치)
└── docs/plans/
    ├── how-we-work.md           ← 작업 방식 설명서
    ├── decisions.md             ← 의사결정 로그 (이미 작성됨)
    ├── ...-design.md            ← 설계 문서 (최신본 유지)
    └── ...-implementation.md    ← 구현 계획 (체크박스로 진행률 추적)
```

## 현재 만들어진 것

**화면 3개:**

- `/login` — 구글 로그인 버튼
- `/` — 코치 목록 테이블 + 우측 슬라이드 패널 (데스크톱)
- `/coaches/[id]` — 코치 상세 (모바일)

**API 1개:**

- `POST /api/sync` — 구글시트 → DB 동기화

**DB 테이블 7개:**

- coaches, courses, coach_courses, coach_memos, sync_logs (사용 중)
- instructors, instructor_courses (3차 대비, 스키마만)

**커밋 10개**, 빌드 정상 통과.

## 5단계: Supabase 연결 + 동기화 (Task 10)

외부 서비스 설정을 완료하고 실제 데이터 동기화를 검증했다.

| 항목 | 상태 |
|------|------|
| Supabase 프로젝트 생성 + DB 마이그레이션 | 완료 |
| Google OAuth 설정 (Supabase + Google Cloud Console) | 완료 |
| Google Service Account 생성 (시트 접근용) | 완료 |
| `.env.local` 키 입력 | 완료 |
| 구글시트 동기화 테스트 | 완료 |

**동기화 과정에서 해결한 이슈:**

| 이슈 | 원인 | 해결 |
|------|------|------|
| Google Sheets API 동작 안 함 | 원본 파일이 .xls (네이티브 스프레드시트 아님) | Google Drive API + xlsx 라이브러리로 전환 |
| 컬럼 매핑 오류 (name="TRUE" 등) | 첫 번째 시트(강사)를 읽고 있었음 | `조교실습코치_일반계약요청` 시트로 변경 + 헤더 기반 재매핑 |
| 로그인 없이 UI에서 데이터 안 보임 | RLS가 인증된 요청만 허용 | 로그인 플로우 정상 확인 후 인증 복원 |
