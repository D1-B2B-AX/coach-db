# Coach DB v2 — 구현 현황 보고

> 2026-03-23 기준 | 설계문서 #1 (스케줄 수집 및 코치 관리) 대비

## 배포

- Production: https://coach-db.up.railway.app
- DB: Railway PostgreSQL
- 파일 저장: Cloudflare R2
- 코치 데이터: 노션에서 45명 이관 완료

## 구현 완료

### 코치 스케줄 입력 (`/coach?token=`)
- 월간 캘린더 + 30분 단위 시간 선택 (07:00~22:00)
- 오전/오후/저녁/종일 빠른 선택
- 가용(초록) / 확정(파랑) / 선택 중(주황) 색상 구분
- 확정 일정 상세 팝업, 나의 스케줄 요약
- 저장 + 마지막 저장 시점 표시
- 모바일 반응형 (하단 저장 바)

### 매니저 대시보드 (`/dashboard`)
- 월간 캘린더 (날짜별 가능 코치 수)
- 날짜 클릭 → 해당일 가능 코치 목록
- 시간대 필터 (전체/오전/오후/저녁)
- 스케줄 입력 현황 (완료/접속만/미입력)
- 30초 자동 새로고침

### 코치 관리 (`/coaches`)
- 코치 목록 (검색, 분야/상태 필터)
- 체크박스 다중선택 + 엑셀 추출
- 코치 등록 (토큰 자동 생성) / 수정
- 코치 상세 — 프로필, 스케줄, 투입 이력 3탭
- 문서 업로드/다운로드 (프로필 내 인라인)
- 코치 삭제 (soft delete + 이메일 확인)

### 인증
- 코치: 64자 토큰 링크
- 매니저: Google OAuth (@day1company.co.kr 제한)

### API
- 24개 엔드포인트 구현 (코치용 4개 + 매니저용 20개)

## 미구현

| 항목 | 우선순위 | 비고 |
|------|---------|------|
| 대시보드 분야/평가/상태 필터 | 높음 | 시간 필터만 있음 |

| 주간 DB 백업 | 낮음 | Railway 자동 백업에 의존 중 |

## 내일 작업 예정

정현님이 공유해주신 프로토타입 HTML(코치 내 스케줄 뷰 + 배정 요청 생성, 4개 버전)을 검토하여 아래 항목을 추가 적용할 예정입니다.

### 코치 스케줄 뷰 (설계문서 #2 범위)
- 확정 일정, 배정요청중(수락/거절), 가용 등록 현황, 지난 이력 표시
- [일정 입력] 탭 전환 연결

### 정현님 보완 버전에서 검토할 항목
- **거절 사유 선택** (일정 불가 / 분야 불일치 / 기타) — 사유에 따라 가용 데이터 처리 분기
- **수락 확인 팝업** — 확정 연결 전 2단계 확인 (모바일 실수 방지)
- **다건 충돌 안내** — 같은 날짜 복수 배정 요청 시 경고 표시 (이중 배정 방지)
- **배정 요청 생성 시 판단 보조 정보** — 코치 선택 시 평점, 총 투입 횟수, 최근 3개월 투입 표시
